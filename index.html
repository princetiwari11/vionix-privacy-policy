VIONIX — Privacy Policy

Last updated: 18 November 2025

Overview

VIONIX (the “App”) is a device optimization, storage inspector and privacy utility. Core features include: system optimization helpers, storage breakdown (apps / images / videos / audio / system), a DNS-based Guard (VPN) mode (DNS-only), a weekly summary, and curated cyber threats headlines.

Data we access / collect and why

The table below describes each permission used by the app (matching the app manifest) and the reason it's required.

Network & remote content

  • Internet (android.permission.INTERNET) — Required to fetch threat headlines, weather data (optional), update checks, and to reach external DNS resolvers when Guard VPN (DNS) mode is active. Networking is performed over HTTPS where possible.

Files, media & storage

  • READ_MEDIA_IMAGES, READ_MEDIA_VIDEO, READ_MEDIA_AUDIO, READ_EXTERNAL_STORAGE — Used only to show local storage breakdown and preview media inside the app Storage views. These are optional and requested at runtime. The app only reads file metadata and sizes to build the storage report. It does not upload your media to any server.
  • WRITE_EXTERNAL_STORAGE (legacy, maxSdkVersion=28) — Present for older device compatibility; not required on modern Android versions that use scoped storage. Not used to upload user content anywhere.
  • MANAGE_DOCUMENTS — Used only to access file picker / document providers if the user explicitly chooses to open files via document UI.

Location

  • ACCESS_FINE_LOCATION, ACCESS_COARSE_LOCATION — Only used if you explicitly enable the optional Weather feature. Location usage is optional: the app will not collect or use location unless you grant it. Location is used locally to request weather forecasts for your area from a 3rd-party weather API (over HTTPS).

Installed apps, usage & system info

  • QUERY_ALL_PACKAGES — Used to enumerate installed apps for the storage & app-management UI (to list installed packages and allow actions such as viewing app storage). This data is processed locally and is not sent to the developer or third parties. This permission is required for the app’s core optimizer/guard features.
  • PACKAGE_USAGE_STATS — Used to show app usage metrics (for storage / performance recommendations). This is an OS-level permission that the user must grant via the Usage Access settings. The app only reads aggregated usage stats (time used, foreground time) to aid optimization; it does not send your usage history to external servers.

Foreground service, notifications & alarms

  • FOREGROUND_SERVICE, FOREGROUND_SERVICE_VPN, FOREGROUND_SERVICE_DATA_SYNC — Required to run the Guard VPN service (VPN-mode) and any background tasks that must remain active while visible to the user. Foreground services show a persistent notification so users know the service is running.
  • POST_NOTIFICATIONS — Required to show app notifications (weekly summary and performance alerts). Android 13+ users must grant this at runtime.
  • SCHEDULE_EXACT_ALARM — Used optionally for scheduled tasks (weekly summary generation or precise widget refresh). Exact alarms are used only when the user opts in to scheduled features.

Other

  • QUERY / Broadcast intents — The manifest contains queries for common intents (MAIN, VIEW, PACKAGE_ADDED) to correctly handle links & package events. These are standard and used only to improve user experience.

What the Guard VPN (DNS-only) does — and does NOT do

The Guard feature uses a local Android VPN service to intercept DNS queries and forward them to the configured DNS resolver. Important facts:

  • DNS-only — Guard does not perform traffic blocking, DPI, or content inspection. It only redirects DNS queries to the selected DNS service.
  • No DNS logging by default — VIONIX does not collect or store DNS queries on the developer servers. DNS resolution is performed by the remote DNS resolver you choose (for example, a public DNS provider). Please review the chosen DNS provider’s privacy policy if you use a third-party resolver.
  • Local routing & privacy — The VPN service runs on the device and the app displays a clear persistent notification while it runs. You may stop the service any time from the app UI or Android system settings.

Data sharing & third parties

VIONIX does not sell or rent personal data. Third parties are only contacted for:

  • Threat headlines (public RSS/feeds) — remote servers that provide the news feed.
  • Weather API — remote provider if you enable weather; this request uses the location you allow.
  • App updates (Google Play / AAB) — update checks use Play infrastructure.

When contacting external services, we use HTTPS and follow best-effort practices to secure data in transit.

Is data encrypted in transit?

Yes — network calls to third-party services and our servers are made over TLS/HTTPS where supported. Any sensitive data transmitted uses encrypted connections.

Retention, deletion & user controls

  • Local-only by default: Most data (installed-app lists, storage size, usage stats) is processed and stored only on the device.
  • Request deletion: If any data associated with your account or device is stored remotely (only happens with explicit user actions), you can request deletion: email princetiwari1@proton.me. We will respond within 30 days.
  • Automatic retention: Temporary activity logs (app’s local activity log, weekly stats) are stored on-device and cleared when you uninstall or when you explicitly reset app data via Settings → Reset All Settings.

Children

VIONIX is not targeted at children under 13. If a parent believes their child’s data has been collected, contact us at princetiwari1@proton.me and we will promptly delete that data.

Security

We take reasonable administrative and technical measures to protect data stored by the app. However, no transmission or storage system is 100% secure — please use the privacy settings available in your device and the app to limit what is shared.
